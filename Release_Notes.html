Purpose

With Ecplise ThreadX RTOS complementing the extensive STM32Cube ecosystem providing free development tools, software bricks, and software expansion packages, STM32 users can also leverage the rich services of Eclipse ThreadX, which meet the needs of tiny, smart, connected devices.

RTOS_AGNOSTIC_U3 (Ecplise ThreadX RTOS Software Expansion for STM32Cube) provides an integration of Eclipse ThreadX in the STM32Cube environment for the STM32U3 series of microcontrollers. Ready-to-run applicative examples are also provided for the NUCLEO-U385RG-Q board, thus reducing the learning curve and ensuring a smooth application development experience with Eclipse ThreadX and STM32U3 MCUs.

The scope of this package covers the following Middlewares: USB Device and Host (Eclipse ThreadX USBX), File System including NOR/NAND memories support (Eclipse ThreadX FileX and Eclipse ThreadX LevelX) and RTOS (FreeRTOS).

Eclipse ThreadX is a professional grade, highly reliable and market proven Middleware suite:

  • Integrated and full featured RTOS.
  • Advanced FS/FTL: fully featured to support NAND/NOR Flash memories. Learn more about [Eclipse ThreadX FileX] here
  • USB Host and Device stacks coming with multiple classes. Learn more about [Eclipse ThreadX USBX] here
  • MIT granted production license: here

Repository structure

The STMicroelectronics Ecplise ThreadX RTOS-Agnostic U3 repository consists of the following repositories:

  • Drivers: contains STM32U3 CMSIS, HAL and BSP drivers
  • Middlewares: contains FreeRTOS, Eclipse ThreadX FileX, Eclipse ThreadX LevelX and Eclipse ThreadX USBX stacks including porting on STM32 hardware

Update history

Main changes

  • First Beta integration of Ecplise ThreadX Middleware in STM32Cube Ecosystem
  • This package integrates the Eclipse ThreadX Middlewares (FileX, LevelX, USBX) with STM32Cube Ecosystem and with RTOS-Agnostic implementation (which means these Middlewares can now be used with FreeRTOS or other RTOSs). It also shows working examples based on this integration. This release is a beta preview release and is subject to some major changes in the future

Known limitations

None

Backward compatibility between Eclipse ThreadX and Azure RTOS Middleware

  • Eclipse ThreadX Middleware uses the same APIs as Azure RTOS Middleware. Modifications on the code have been operated by STMicroelectronics to allow FileX, LevelX and USBX Middlewares to be RTOS-Agnostic (usable with FreeRTOS or other RTOSs)
